1¿Qué es una API? (Explicación básica)

  • Una API es como un camarero digital que lleva tu pedido (request) al servidor y trae de vuelta la respuesta.
  • Cliente: tú o tu ordenador, pidiendo información.
  • Petición HTTP: el pedido que se envía al servidor.
  • Respuesta HTTP: lo que el servidor devuelve.
  • Salida: normalmente llega en formato JSON (a veces XML o HTML), que Make puede leer y procesar.

2Tipos de peticiones: GET vs POST

  • GET – “Consigue datos”
    • Se usa para leer información que ya existe.
    • Ejemplo: buscar mascotas por ID, raza o peso en una reserva natural.
    • Piensa en GET como “¿Qué tienes aquí? Muéstramelo”.
  • POST – “Envía / Publica datos”
    • Se usa para crear o actualizar información en la base de datos.
    • Ejemplo: crear la ficha de una nueva mascota o subir una foto.
    • Piensa en POST como “Toma esto y guárdalo”.

3Caso 1 – Conectar un servicio que no existe en Make (Polygon.io)

  • Cuando un servicio NO está en el catálogo de Make (por ejemplo, Polygon.io):
  • 1. Elegir módulo: usa el módulo genérico HTTP – Make a request.
  • 2. Documentación: busca la documentación de la API (por ejemplo “polygon.io documentation API”).
  • 3. Función deseada: localiza el endpoint que necesitas, como “daily open close” (GET).
  • 4. Parámetros: en la documentación rellena el símbolo del activo (ej. AAPL) y la fecha a consultar para que te construya el ejemplo de llamada.

4Configurar URL y API Key (Caso 1)

  • Obtener la URL: copia el enlace completo que genera la documentación con tus parámetros.
  • Asegúrate de que la URL incluya tu API Key. Si no se añade sola, pégala al final como parámetro.
  • Pegar en Make: en el módulo HTTP, coloca esa URL en el campo “URL”.
  • Parsear respuesta: activa “Parse response → Yes” para que Make convierta el JSON en campos legibles.

5Usar los datos obtenidos (Caso 1)

  • Ejecuta el módulo para hacer la prueba: verás un JSON con precios de apertura, máximo, mínimo, volumen, etc.
  • Conecta un módulo siguiente (por ejemplo una base de datos o Google Sheets).
  • Usa el mapeo de Make para arrastrar los campos obtenidos (símbolo, fecha, precios) hacia las columnas donde quieras guardarlos.

6Caso 2 – Usar una función oculta en un módulo existente

  • Algunas apps sí están en Make (ej. Zero Code Kit) pero no ves un botón directo para la función que quieres.
  • 1. Agregar módulo: añade el módulo de la app (Zero Code Kit).
  • 2. Elegir “Make an API call”: en vez de una acción prehecha, selecciona la opción “Make an API call”.
  • 3. Conectar tu cuenta: usa la API Key de la app si aún no la conectaste.
  • 4. Documentación: busca la función concreta (por ejemplo “compare string” o “compare to string”) y revisa método (POST) y estructura del body.

7Configurar la llamada POST (Caso 2)

  • Endpoint (URL): en “Make an API call” solo pones la parte final, por ejemplo /compare-to-string; la base URL ya la conoce Make.
  • Método: selecciona POST.
  • Header: añade Content-Type con valor application/json para indicar que envías JSON.
  • Body: pega el JSON de la documentación, con los campos que la API espera (por ejemplo string_one y string_two).

8Ejecución y actualización de registros (Caso 2)

  • Disparador: una tabla o base de datos donde tienes las dos frases a comparar.
  • Respuesta: la API devuelve valores como similaridad y distancia entre textos.
  • Actualizar: con un módulo final, guarda esos valores en la fila correspondiente, junto a las frases originales.

9Analogía gamer y consejos finales

  • Aprender bien JSON es clave si vas a usar APIs de forma intensiva.
  • Imagina Make.com como un mando de videojuegos:
    • Los módulos normales son los botones X, Y, A, B (acciones preconfiguradas).
    • Los módulos HTTP y “Make an API call” son el panel de comandos especiales para movimientos que no vienen de fábrica.
  • GET = “¿Cuántas monedas tengo? Muéstrame mis datos”.
  • POST = “Guarda la partida / crea personaje / sube estos datos”.
  • La documentación de la API es el manual de trucos: ahí está el código exacto que debes introducir para que todo funcione.

10Validación con fuentes técnicas oficiales

  • El módulo HTTP de Make permite conectar con cualquier servicio web mediante GET, POST, etc., incluso sin integración oficial.
  • Muchos conectores de Make incluyen la acción “Make an API call” para acceder a endpoints no cubiertos por módulos estándar.
  • Polygon.io ofrece el endpoint Daily Open/Close que devuelve datos financieros de un símbolo y fecha concretos en JSON.
  • JSON (JavaScript Object Notation) es el formato estándar ligero para intercambio de datos en APIs modernas.
  • Consulta siempre la documentación oficial de:
    • Make.com – Módulo HTTP y “Make an API call”.
    • Polygon.io – endpoints financieros.
    • Introducción a JSON (por ejemplo JSON.org).
